## 백엔드 아키텍처 (plan-eng-review 2026-06-22, 스코프 A = 마이그레이션 MVP)

스코프: 입력 로그 제출/조회 + 데일리 리더보드 + 콜드스타트 봇 고스트 + 인앱 포인트 장부.
인증/인앱화폐 소비는 훅만(사업부 실무 게이트 대기). 결정 6건:

| # | 결정 | 선택 | 근거 |
|---|------|------|------|
| B1 | 호스팅/DB | **Supabase (관리형 Postgres)** | boring by default. lock-in 없음(pg_dump로 직접구축 이전 쉬움). API 자동생성. |
| B2 | 화폐 장부 | **오프체인 (Postgres row)** | 닫힌 경제라 온체인 투명성 불필요. 단순·무가스. 온체인은 게임코인 단계 옵션. |
| B3 | 점수 검증 | **이상치 필터링 (D11 유지)** | 닫힌 경제(현금가치 0)라 부정 동기 약함. 서버 리플레이는 실물 보상 시 승격. |
| B4 | 콜드스타트 봇 | **클라(브라우저)에서 녹화한 봇 로그를 서버가 저장/배포** | 외부검토 #2 수용: 서버 sim 실행은 런타임 차(Deno≠브라우저)로 결정론 깨질 위험 → sim은 클라(단일 런타임)에서만. |
| B5 | 서버 역할 | **dumb store (sim 실행 안 함)** | B4 귀결: 서버는 JSON 로그 저장/조회/정렬만. sim 코어는 클라 전용 패키지(렌더와 분리는 유지). |
| B6 | 클라 연동 | **기존 KVStore 인터페이스 뒤 원격 어댑터** | 재작성 아님. localStorage 폴백 구조가 백엔드 다운 시 로컬 degrade 제공. |

마이그레이션 원칙: 표준 Postgres + 표준 SQL만 사용. B5(dumb store)로 Supabase 고유표면(Edge Function·pg_cron) 의존이 줄어 lock-in 더 약함(외부검토 #7).

외부검토(Claude 서브에이전트) 흡수 — 설계 반영/TODO:
- **#3 버전 분절:** 리더보드·고스트를 **SIM_VERSION으로 파티션**(클라 ghostKey가 이미 버전 포함 → 서버 키에도 적용). v1/v2 클라가 같은 보드에 안 섞임. 첫 sim 패치 전 필수.
- **#4 리더보드 오염:** 닫힌 경제라도 동기는 '1등'. 이상치 필터(B3)에 더해 **의심 로그는 고스트 배포 전 격리**, top-N만 선택적 리플레이 검증은 실물 보상 시 승격(TODO).
- **#5 오프라인 머지:** 백엔드 다운 중 로컬 기록 → 복구 시 **서버 우선/멱등** 병합(이중 적립 방지). MVP는 포인트 미구현이라 읽기 degrade 위주(TODO).
- **#6 봇 시드 커버리지:** 클라 녹화 봇이라 cron 부담↓. 단 데일리 시드별 봇 로그가 비지 않게 **시드 범위 미리 녹화 + '전날 봇 폴백'** 보장.

```
        ┌─────────────────────────────────────────┐
        │  WebView 클라이언트                        │
        │  sim 코어 ◄── 공유 패키지(B5) ──► 서버      │
        │  KVStore 인터페이스(B6)                    │
        │   ├ localStorage (폴백 = 로컬 degrade)     │
        │   └ 원격 어댑터 ──────┐                    │
        └──────────────────────│────────────────────┘
                               ▼  제출(로그+거리) / 조회
        ┌──────────────────────────────────────────┐
        │  Supabase (B1)                             │
        │  ├ 로그 저장 (이상치 필터 B3)               │
        │  ├ 리더보드: (seed, distance DESC) 인덱스   │
        │  ├ 포인트 장부 (오프체인 row, B2)           │
        │  └ cron: 서버 sim으로 봇 고스트 생성 (B4)   │
        └──────────────────────────────────────────┘
```

백엔드 신규 코드패스 테스트 (구현 시 동반):
- 로그 제출: happy / 손상·버전불일치 거부(parseLog 재사용) / 이상치 거부
- 리더보드 조회: happy / 빈 결과 / 시드 불일치
- ★ CRITICAL: 백엔드 다운 → 로컬 degrade (게임 안 깨짐)
- 봇 로그 배포: 시드별 봇 존재 / 빈 시드 시 전날 폴백
- 버전 파티션: v1 로그가 v2 보드에 안 섞임 (SIM_VERSION 키 분리)
- (서버 sim 골든 검증은 B4 변경으로 불필요 — sim은 클라 단일 런타임)

성능: 리더보드 (seed, distance DESC) 복합 인덱스 필수. 고스트 로그는 데일리 시드 파티션 + 오래된 시드 정리(보존 정책, TODO).
