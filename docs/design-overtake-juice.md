# 설계: 제침 순간 연출 (Overtake Juice) — 계단식

작성일: 2026-07-24
출처: office-hours (Builder 모드) 세션
관련: `docs/retention-update-plan.md` §2.1
상태: 설계 확정 (구현 대기 — 코드 없음)

---

## 1. 문제 & 목표

- **배경:** 라이브 데이터에서 74% 판이 0제침이었고, 필드 "실력 사다리"(§2.4, `517af5f`)로
  median 유저가 판당 ~3제침을 하게 됐다. 이제 제침이 자주 일어나므로, **그 순간의 연출**이
  리텐션 레버로서 효과가 커진다.
- **목표:** 고스트가 "이미 달린 실경쟁자"로 느껴지고, 제칠 때 "내가 진짜 사람을 제쳤다"는
  쾌감이 살아나게 한다.
- **제약:** 순수 렌더(SIM_VERSION 무관, GRAC 밸런스 신고 대상 아님). 모바일 토스 웹뷰 퍼포
  안전. 닷지(점프) 방해 금지.

## 2. 지금 이미 터지는 것 (중복 제안 방지)

제침(`GameScene.ts:3089` `!wasFinished && g.finished`) 시:
- `"고스트 제침!"` 플로팅 팝업 (보라 `#b39ddb`)
- `sfx-overtake` 사운드
- 제친 고스트의 돌진 → `ghost-collapse` 고꾸라짐 (`9e04cb4`)
- 고꾸라지는 고스트 머리 위 이모션 말풍선 (`showGhostEmotion`)

**아직 없음:** 팝업 닉네임+거리, 화면 플래시/파티클, 카메라 반응, HUD 실시간 순위 변동,
연쇄 크레셴도, 격추 로그.

## 3. 설계 원칙 (확정)

1. **계단식(Tiered).** 매 제침마다 전부 터뜨리지 않는다. 연출 강도 = 사건의 무게에 비례.
   제침이 판당 ~3회(최대 8회) 발생하므로, 무차별 풀연출은 (a) 피로·싸구려화, (b) 모바일
   프레임 드랍(이미 씨름한 문제), (c) 점프 중 방해로 인한 억울사 를 부른다.
2. **슬로모 금지 → 렌더 대체.** 진짜 슬로모는 고정 타임스텝 결정론 sim·고스트 lockstep·
   리더보드 공정성을 흔든다. "큰 순간"은 **플래시 + 기존 `punchZoom()` + 파티클**로 표현한다
   (시간 정지 없음).
3. **실유저/봇 배지 없음 (의도적 비목표).** 모든 고스트를 균일하게 대우 = "전부 사람"
   환상 유지. 봇은 이미 실유저형 랜덤 닉(`deterministicNickname`)을 달고 있어 구분이 오히려
   환상을 깬다. → 닉네임 팝업은 봇/실유저 동일 처리.

## 4. Tier 정의 (트리거 + 연출)

| Tier | 트리거 (제침 순간 판정) | 연출 |
|------|------------------------|------|
| **1** 일상 | 단일 제침 (streak==1). 대부분의 제침 | 닉네임+거리 팝업 · 고스트 힐끗+고꾸라짐(기존) · sfx 피치 1계단 · HUD 순위 틱 |
| **2** 고조 | 연쇄 2번째(streak==2) **또는** 처음 top-3 진입 | Tier1 + 짧은 화면 플래시 · 파티클 버스트 · 가벼운 `punchZoom` · 사운드 크레셴도 |
| **3** 하이라이트 | 연쇄 3+ (streak>=3) **또는** 1위 탈환(rank==1 신규) | Tier2 강화(큰 플래시·강한 punchZoom) + 격추 로그 플러시 |

**streak 정의:** 롤링 윈도(예: 2.5초 = `SIM_FPS*2.5`프레임) 안의 연속 제침 수. 마지막
제침 프레임 + 카운터로 추적. 윈도 밖이면 streak=1로 리셋.

**rank:** 이미 계산됨 — `rank = ghosts.length - overtakenLive + 1`. Tier 판정에 재사용.

## 5. 연출별 구현 노트 (구현 시 참조)

- **닉네임+거리 팝업:** 기존 `popup()` 확장. 텍스트를 `"{nick} 제침!  {dist}m"`로.
  닉 출처: 제친 고스트의 `record.log.meta.nickname` (없으면 `deterministicNickname`).
  거리 = 그 고스트의 최종 거리(`ghostDistances[i]`).
- **고스트 힐끗:** 고꾸라지기 직전 1프레임 이쪽을 보는 포즈. `ghost-collapse` 진입 직전
  스프라이트 flipX 토글 or 전용 프레임 1장. "사람이었다" 각인, 저비용.
- **HUD 실시간 순위 틱:** 현재 순위 인디케이터를 HUD에 상시 표시, 변동 시 "7위→6위" 팅
  스케일 펄스. rank 변수 재사용.
- **사운드 크레셴도:** 연쇄 제침마다 `sfx-overtake` detune 상향(피치 계단). streak 카운터에
  비례. (점프 detune 패턴 참고: `SFX_JUMP_DOUBLE_DETUNE`.)
- **플래시:** 짧은(≤120ms) 반투명 오버레이 페이드. 전체화면이되 alpha 낮게 — 닷지 시야 방해
  최소. Tier2 약, Tier3 강.
- **파티클:** 제친 고스트 위치에서 짧은 버스트. 개수 상한(모바일). 피버 중엔 생략(가독성).
- **카메라:** 기존 `punchZoom()` 재사용 — Tier2 약, Tier3 강. 슬로모 대체재.
- **격추 로그(Tier3):** 화면 구석에 방금 제친 닉 잠깐 쌓임 ("시안여우-42 ✕"). 자동 페이드.

## 6. 리스크 & 완화

| 리스크 | 완화 |
|--------|------|
| 모바일 프레임 드랍 (플래시·파티클·punchZoom) | 계단식으로 무거운 효과를 Tier2/3에 한정. 파티클 상한. 피버 중 생략. |
| 결정론 훼손 | 슬로모 배제. 모든 연출은 렌더 전용, sim 시계·입력·고스트 재생에 무접촉. |
| 닷지 중 방해로 억울사 | 플래시 alpha 낮게, 슬로모 없음, 화면 중앙 가림 금지. |
| 팝업 폭주(8제침 판) | 팝업 큐 상한 + 겹칠 때 병합("×3 제침"). |

## 7. 비목표

- 실유저/봇 배지·구분 (§3.3 — 의도적으로 안 함).
- 진짜 슬로모/타임스케일 조작.
- sim 로직·밸런스 변경 (렌더 전용 유지).

## 8. 구현 순서 (권장)

1. **Tier 1 먼저** — 닉네임+거리 팝업, HUD 순위 틱, sfx 피치, 고스트 힐끗. 빈번한 케이스를
   다 커버하고 저위험. 이것만으로도 체감 크게 상승.
2. **streak 추적 + Tier 2** — 플래시·파티클·punchZoom.
3. **Tier 3** — 1위 탈환/연쇄 피날레 강화 + 격추 로그.

각 단계 후 `npm run dev` 플레이테스트로 강도 튜닝. 상수로 빼서(플래시 alpha/시간, 파티클 수,
punchZoom 배율, streak 윈도) 조정 가능하게.

## 9. 열린 질문 (튜닝/후속)

- streak 윈도 길이(2.5초?) — 플레이테스트로.
- HUD 순위 인디케이터 위치 — 기존 HUD 레이아웃과 충돌 확인 필요.
- 격추 로그가 화면을 어지럽히는지 — Tier3 한정이라 빈도 낮지만 실측 필요.
