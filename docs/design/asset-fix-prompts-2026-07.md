# 에셋 통합 수정 프롬프트 (2026-07) — Sonnet 4.6용

앞선 에셋 통합(패널·배경·연료통) 후 다수 결함 발생. 이 문서는 **원인 분석 + 구체 수정 지시**를
담은 실행 스펙이다. Sonnet 4.6은 이 문서를 읽고 순서대로 고친다.

## 공통 원칙 (반드시 준수)

- **렌더 전용 변경만** — `assets/`, `scripts/prep-*.py`, `src/render/GameScene.ts` 만 손댄다.
  `src/sim/**`·`SIM_VERSION`·고스트 로그 스키마·충돌 판정은 **절대** 건드리지 않는다.
  (부유 모션 등은 "표시 좌표"만 바꾸고 시뮬 좌표 `p.x/p.y` 는 그대로 둔다.)
- **파이프라인**: 원본 `assets/images/<cat>/<name>-src.png` → `scripts/prep-*.py` →
  가공본 `assets/game/<name>.png` → GameScene `import` + `this.load.image`.
- 각 작업 후 **`npx tsc --noEmit` 통과** + prep 스크립트 재실행(`python3 scripts/prep-*.py`) +
  인게임 육안 확인. 텍스처 캐시 때문에 브라우저는 **하드 새로고침** 필요.
- 색/해상도 문제 대부분은 에셋 화질이 아니라 **표시 방식(nineslice 인셋·강제 종횡비·스케일)** 이 원인.
  → 아래 근거 데이터를 신뢰하고, 무작정 에셋을 다시 만들지 말 것.

## 문제 1 — 연료통(fuel-can): 너무 큼 + 배경 안 지워짐

**증상**: 아이템이 크게 붙어 게임이 쉬움. 캔 주위에 **네모난 배경 박스**가 남아 투명 처리가 안 됨.

**원인 (측정 근거)**:
- 크기: `src/render/GameScene.ts:182` `const FUEL_ART_SIZE = 100;` (주석에 "52→100 2배"). 과대.
- 배경: 소스 `assets/images/items/fuel-can-src.png` 는 **알파 없는 RGB(1024×1024)** 이고
  배경이 **밝은 회색(223~236) + 흰색(253~255) 체커무늬로 이미지에 그려져** 있다(진짜 투명 아님).
  `scripts/prep-ui.py`의 `_remove_white_bg(thresh=238)` 는 **흰색만** 지우고 **회색 체커는 남긴다**
  → 결과 `assets/game/fuel-can.png` 의 코너 alpha=255, 불투명 픽셀 93%. 이게 네모 박스로 보임.

**수정 지시**:
1. `scripts/prep-ui.py` `prep_fuel_can()` 의 배경 제거를 **체커 전체(회색+흰색)** 를 지우게 바꾼다.
   - 방법 A(권장, 안전): **네 코너에서 flood-fill** — 시작색과 유사한(밝고 무채색: `min(r,g,b)>205`
     이고 채도 `sat<0.10`) 연결 영역만 α0. 캔 본체(어두운 남색)·시안 글로우는 색이 달라 보존됨.
   - 방법 B(간단): 픽셀별로 `min(r,g,b) > 205 && sat < 0.10 → α0`. (흰 253+ 회색 223+ 모두 포함)
   - 제거 후 캔 외곽 파란 글로우가 잘리지 않게 알파 경계 1~2px GaussianBlur 페더링.
   - 이후 기존대로 bbox 트림 → 정사각 캔버스 중앙 배치.
2. `GameScene.ts:182` `FUEL_ART_SIZE = 100` → **`50`** (현재의 절반).
3. `python3 scripts/prep-ui.py` 재실행 → `assets/game/fuel-can.png` 갱신.

**수용 기준**: 인게임 연료통이 **투명 배경**으로 캔만 보이고(네모 박스 없음), 크기가 지금의 절반,
파란 글로우가 배경에 사각으로 남지 않는다.

## 문제 2 — 연료통 부유(bob) 모션 추가

**증상**: 아이템이 정적이라 밋밋함. 위아래로 살짝 떠다니는 모션 원함.

**원인**: 표시 로직(`GameScene.ts:2937~2942`)이 시뮬 좌표를 그대로 그림 — 애니메이션 없음.

**수정 지시** (렌더 전용, 결정론 안전):
- `syncVisuals` 의 연료통 위치 지정부에서 **표시 y에만** 사인 오프셋을 더한다.
  시뮬 좌표 `p.x/p.y` 는 불변 → 충돌 판정 영향 없음.
```ts
// 2941 근처
if (p.active) {
  const t = this.time.now / 1000;           // 벽시계(시뮬 RNG 아님)
  const bob = Math.sin(t * 2.2 + i * 1.3) * 4; // 진폭 4px, 아이템마다 위상차
  c.setPosition(toScreenX(p.x), toScreenY(p.y) + bob);
}
```
- (선택) 살짝 회전/스케일 맥동을 더해도 됨: `c.setScale(1 + Math.sin(t*3)*0.03)` 수준.

**수용 기준**: 연료통이 부드럽게 위아래로 떠다니고, 먹는 위치(충돌)는 기존과 동일(시뮬 불변).

## 문제 3 — 상단 즉석 랭킹 칩(rank-hud): 뭉개져 깨짐

**증상**: 상단 4칸 랭킹 칩이 픽셀·색 다 깨지고 프레임이 뭉개짐.

**원인 (근거)**: `GameScene.ts:883~916` nineslice 인셋이 **표시 크기보다 큼**.
- 표시: `RP_H = 30`, `RP_W = 218`.
- 인셋: `RP_INS = { l:40, r:40, t:32, b:32 }`.
- 세로: **상+하 인셋 = 64 > 표시 높이 30** → nineslice 코너가 서로 겹쳐 **크러시**(깨짐).
- 소스: `panel-rank-hud` 952×183, `panel-rank-hud-gold` 957×149.

**수정 지시**:
- 인셋을 표시 크기 안에 들어오게 축소. 규칙: `t+b ≤ RP_H`, `l+r ≤ RP_W`, 코너 장식만 보존.
  예: `RP_INS = { l:24, r:24, t:10, b:10 }` (높이 30에 상하 20 → 중앙 10 확보).
- 그래도 세로가 빡빡하면 `RP_H` 를 36~40으로 키우고 상단 y·컨테이너 배치 재조정.
- 인셋은 **소스별로 실제 네온 코너 두께**를 재서 정한다(가로 프레임이라 t/b는 얇게).
  (측정 팁: 이미지 상/하단에서 네온 라인이 끝나고 평평해지는 지점까지의 px.)

**수용 기준**: 4칸 칩이 네온 프레임 그대로(코너 안 뭉개짐), YOU=골드/고스트=시안, 텍스트가 칸 안에 정렬.

## 문제 4 — 주간 랭킹 패널(weekly) & 튜토리얼 패널(tutorial): 종횡비 파괴 + 텍스트 어긋남

**증상**: 주간 랭킹 패널·튜토리얼 오버레이가 깨지고 크며 텍스트와 조화가 안 됨. (튜토리얼은 **첫
피버 발동 시 1회만** 떠서 확인이 어려움 — `ga:fever-tutorial` localStorage 삭제하면 재현 가능.)

**원인 (근거)**: 두 소스 모두 **세로형 + 내부 레이아웃이 그려진(baked) 패널** 인데
**가로 박스로 강제 nineslice** 해서 내부 구획(헤더 탭·히어로 박스·행 구분선)이 뭉개짐.
- 주간: 소스 `panel-weekly` **758×1021(세로)** → `GameScene.ts:937` nineslice **440×250(가로)**.
  게다가 행 텍스트 좌표(`-190, -42+i*24 …`, `:963~992`)는 **옛 사각 패널 기준** → 아트 구획과 불일치.
- 튜토리얼: 소스 `panel-tutorial` **800×1016(세로)** → `GameScene.ts:1089` nineslice **420×180(가로)**.
  텍스트는 화면 중앙 기준 배치라 프레임 구획과 안 맞음.

**핵심 개념**: 이 패널들은 nineslice로 "늘려 쓰는 프레임"이 아니라 **내부 디자인이 고정된 그림**이다.
→ **원본 세로 종횡비를 유지한 정적 `Image`** 로 얹고, **텍스트를 아트의 baked 구획에 맞춰** 배치해야 한다.
(정말 nineslice를 쓰려면, 늘어나도 되는 순수 테두리 여백만 인셋으로 잡고 내부는 안 늘어나게 해야 함.)

**수정 지시 (주간 패널)**:
1. `wkBg` 를 nineslice → **`this.add.image(0,0,"panel-weekly")`** 로 교체하고,
   세로 종횡비 유지로 크기 지정: 예 폭 `PW=360` → 높이 `PW*1021/758 ≈ 485`. `setDisplaySize(360, 485)`.
2. 컨테이너 위치·텍스트 y좌표를 새 패널 구획에 맞춰 재배치:
   - 상단 마젠타 탭 = "주간 랭킹 · 7일 누적" 제목.
   - 그 아래 시안 히어로 박스 = 이번 판 거리(`gameOverDistText`).
   - 노란 하이라이트 바 = (나) 행 강조.
   - 아래 행 구획선들 = `weeklyRowTexts` 5행(구분선은 아트에 있으니 코드로 그리지 말 것).
   - 최하단 캡슐 = "TAP TO RESTART" 힌트.
3. 텍스트 폭이 패널을 넘치지 않게 `fontSize`·`wordWrapWidth`·`x` 좌표 조정.

**수정 지시 (튜토리얼 패널)**:
1. `ftBg` 를 nineslice → 세로 종횡비 유지 `Image` 로 교체(예 폭 300 → 높이 `300*1016/800≈381`).
2. FEVER!/설명/서브 텍스트를 패널 세로 구획 안쪽으로 재배치, `align:center`, 폭 넘침 방지.
3. 재현: DevTools에서 `localStorage.removeItem('ga:fever-tutorial')` 후 피버 발동으로 확인.

**수용 기준**: 두 패널이 **세로 비율 유지**로 선명하게 뜨고, 텍스트가 아트 구획과 정확히 맞으며,
화면을 벗어나지 않는다. 코드로 그리던 구분선/배경 사각형은 제거.

## 문제 5 — 배경 원경 레이어: 코드 스카이라인과 겹쳐 지저분/깨짐

**증상**: 일간/주간과 함께 배경도 "제대로 된 게 없음". 픽셀·색 깨짐, 지저분함.

**원인 (근거)**: `GameScene.ts:2096~2132` 에서 **코드로 그린 스카이라인(`drawSkyline` ×2)** 위에
**이미지 원경 타일(`bg-buildings-far` α0.85)** 을 겹쳐 얹음 → **두 겹의 스카이라인이 충돌**해 노이즈/난잡.
- `bg-bridges-far`·`bg-bridges-curved-far` 는 α0.0(숨김)이라 바이옴 스왑이 안 걸린 듯.
- 타일 스케일·높이는 수학상 맞지만(2048×tileScale), 코드 스카이라인과의 **이중 노출**이 핵심.

**수정 지시**:
1. **레이어 정리** — 이미지 원경을 주 배경으로 쓸지, 코드 스카이라인을 유지할지 택1.
   - 권장: 이미지 far 타일을 쓰는 구간에선 **코드 스카이라인 alpha를 크게 낮추거나 숨김**
     (`g1/g2` 컨테이너 alpha ↓), 이미지 타일 alpha는 0.9~1.0로.
2. **바이옴 스왑 배선** — 바이옴/거리 구간에 따라 buildings ↔ bridges(직선/곡선) 중 하나만 alpha=1,
   나머지 0으로 크로스페이드. (시드 결정론 LCG로 곡선/직선 택1, sim RNG와 분리.)
3. **seam 확인** — `prep-bg.py` 의 좌우 8px 페이드가 실제로 이음매를 없애는지 스크롤 중 확인.
   안 되면 `EDGE_FADE_PX` 상향 또는 좌우 크로스블렌드로 개선.
4. 색: 타일에 biome 틴트를 과하게 주지 말 것(원 팔레트 살리기). 필요 시 약한 곱연산 틴트만.

**수용 기준**: 달릴 때 원경이 **한 겹으로 깔끔하게** 흐르고(이중 스카이라인 없음), 이음매·색 이상 없음,
바이옴 전환 시 배경이 buildings↔bridges 로 자연스럽게 바뀐다.

## 문제 6 — HP바 우측 상단이 여전히 빔 (하트 근처는 개선됨)

**증상**: 트랙 추가로 하트 근처는 나아졌으나 **우측 상단**이 아직 어둡게 비어 보임.

**원인 (측정 근거)**: `assets/game/hp-frame.png` 내부 구멍의 **윗변이 대각선**(좌 0.18h → 우 0.27h,
하트 우측은 다시 0.18h). 현재 fill/track 높이는 `barH*0.82`(중앙 정렬 → 세로 0.09~0.91h 커버)라
대부분 덮지만, **경계에서 얇은 슬리버**가 남을 수 있고, HMR 시 **텍스처 캐시**(`hp-fill-grad`)가
옛 높이로 남아 갱신이 안 될 수 있음. (관련: `GameScene.ts:~795 fillTexH`, `~820 hpTrack`.)

**수정 지시**:
1. fill/track 세로 커버를 **구멍 최대치보다 확실히 크게** 올린다: `fillTexH` 계수 `0.82 → 0.95`
   (넘치는 부분은 불투명 테두리가 가리므로 안전).
2. 텍스처 캐시 무효화: 캔버스 텍스처 키를 버전 접미사로 바꿔 재생성 강제
   (`"hp-fill-grad"` → `"hp-fill-grad2"`, 관련 `exists()`/`createCanvas`/`image` 키 3곳 모두 교체).
3. 트랙 이미지도 동일 텍스처·높이라 자동 반영되는지 확인.

**수용 기준**: 만땅~소진 전 구간에서 초록 게이지 위/아래·우측 상단에 어두운 틈이 안 보이고,
빈 구간은 어두운 트랙으로 균일하게 보인다.
