# 개선 백로그 (비용 오름차순)

> 2026-06-27 정리. 14개 요청을 **비용(작업량 + 위험도)** 기준으로 묶었다.
> 핵심 경계: **sim 변경 =** `SIM_VERSION` **업 = 기존 고스트/리더보드 무효화**(론칭 전이라 무료지만 묶어서 처리). **렌더 변경은 버전 무관, 언제든 자유.**
>
> 적용은 위(싼 것)부터 하나씩.

## ★ 최우선 작업 (2026-06-28 지정) — "다음 마일스톤"

> 사용자 지정 우선순위. 아래 4개를 **다른 무엇보다 먼저** 처리한다. 전부 *렌더/에셋/백엔드 폴리시*라
> sim 변경(버전 범프) 없이 진행 가능 — 완성도를 끌어올려 "한 판 더"의 첫인상을 만든다.

1. **주인공 캐릭터 고도화** — `player-rider` 컷 품질↑(ride/jump/jump2/hit/dead), 글로우/스카프/트레일 정돈.
   에셋 프롬프트는 `asset-guide.md §5.1`(+ §5.1 ★player-jump2). 2단 점프 컷 적용 시 현재 코드 회전 스톱갭(A-4) 대체. 남캐에서 귀엽고 힙한 여캐와 오토바이로 변경
2. **고스트 고도화** — 달리는 고스트를 **후드 유령 컨셉**으로 재생성(주인공과 세계관 통일, 실루엣은 구분).
   프롬프트 `asset-guide.md §5.2A`. 화질·프레임 연속성도 함께 점검. 고스트들도 랜덤이긴 하지만 에셋을 좀 다양하게 뽑을까 ?
3. **UI 에셋 부착** — hp바·랭킹 패널·튜토리얼·게임오버 패널을 코드 Rectangle → 네온 에셋(9-slice)으로 교체.
   프롬프트 `asset-guide.md §5.7`(hp-bar) + `§5.7B`(rank/tutorial/gameover). 글자는 코드 텍스트 유지.
4. **이번 주 리더보드(주간 거리 랭킹)** — 게임오버 패널/별도 탭에 **최근 7일 누적·최고 거리** 랭킹 표시.
   백엔드 집계(주간 윈도우 쿼리) 필요 → `remoteStore` 확장. sim 무관(기록 메타만 사용).

> **참고:** 2층(멀티레인) 장애물은 **작은 화면에서 가독성/난이도 우려가 커 보류**(아래 5-1 참조). 최우선 아님.

## Tier 0 — 거의 공짜 (렌더/상수, sim 무관, 결정 불필요)

### 0-1. 고스트 8명 제한 ✅ 적용 후보 1순위

- **What:** `GHOST_TOP_N` 15 → 8. 저장·원격 쿼리·렌더 풀 모두 이 상수를 공유하므로 한 곳만 바꾸면 됨.
- **Why:** 화면이 8명 정도일 때 군집 가독성이 좋고, 봇 8개 정책(0-? / Tier2)과도 수가 맞는다.
- **경계:** `src/ghostStore.ts` 상수 1줄. sim 무관 → 버전 업 없음.
- **주의:** `GHOST_TOP_N` 하드코딩 테스트가 있으면 같이 수정.

### 0-2. 오글거리는 랜덤 말풍선

- **What:** 검은 사각 말풍선 + 흰 글씨를 일정 간격마다 랜덤으로 표시.
  - 시작 문구 3종: `지금 내가 걷는 길은 이전에 {n}명의 러너들이 달린 길이지`(n=현재 고스트 수),
    `너와 함께라면 지구 끝까지라도 가보겠어!`, `하루 종일 할 수도 있어`.
- **Why:** 세계관·정서 몰입. 저비용 고효과 분위기 장치.
- **경계:** 렌더 전용(`GameScene`) — 타이머 + 텍스트 풀. sim 무관.

### 0-3. 고스트 점프 시 프레임 정지

- **What:** 고스트가 공중(`g.sim.state.player.y > 임계값`)일 때 `ghost-run` 애니를 멈추고 고정 점프 프레임으로.
- **Why:** 지금은 공중에서도 달리는 모션이라 어색. 플레이어 점프 컷과 일관.
- **경계:** 렌더 전용 — 고스트 스프라이트 anim pause/frame. sim 무관.

## Tier 1 — 저렴 (렌더 전용, 약간의 로직/연출)

### 1-1. 고스트가 죽을 때 "제치는" 이모션

- **What:** 고스트가 기록 끝(=쓰러짐)에 도달할 때, 그 위에 이모지/말풍선 이모션 표시(예: 손 흔들기, 😵, "먼저 갈게~").
- **Why:** 이미 `고스트 제침!` 팝업은 있으나, *그 고스트 머리 위*에 붙는 연출이 추월의 쾌감을 키운다.
- **경계:** 렌더 전용 — collapse 트리거 시점에 이모션 스프라이트/텍스트. 0-2 말풍선 인프라 재사용 가능.

### 1-2. 재시작 시 고스트 정보 최신 재요청

- **What:** "탭하여 재시작" 시 캐시(`this.remoteRuns`)가 아니라 **새로 fetch한 원격 기록**으로 고스트 구성.
- **Why:** 현재 `startRun`은 직전 캐시로 즉시 고스트를 깔고 백그라운드 갱신만 함 → 방금 올라온 남의 기록이 다음 판에 안 보일 수 있음.
- **경계:** 렌더/네트워크 전용. 비동기 순서(로딩 중 임시 고스트 → 도착 시 교체) 처리 필요. sim 무관.

### 1-3. 오토바이 네온 트레일(속도감)

- **What:** 오토바이 뒤에 하늘색 네온 라인/도트 잔상이 반짝이며 따라붙는 이펙트.
- **Why:** 속도감/스타일 강화. 피버 때 강하게 하면 보상감↑.
- **경계:** 렌더 전용 — Graphics/파티클. sim 무관. 속도(`s.speed`)에 비례시키면 자연스러움.

### 1-4. 2단 점프 80도 꺾임 — 에셋 프롬프트

- **What:** 2단 점프 시 ~80도로 더 꺾인 라이더 컷 에셋 **프롬프트만** 작성(`asset-guide.md`).
- **Why:** 2단 점프의 타격감/역동성. 프롬프트는 거의 공짜, 에셋 적용(텍스처 스왑)도 소규모.
- **경계:** 프롬프트=문서. 적용 시 렌더 전용(점프 상태별 텍스처 분기에 컷 추가).

## Tier 2 — 중간 (작업량 있음, 일부 결정/검증 필요)

### 2-1. 불기둥 → 코드 드로우 화염분수 + 오염수 장애물

- **What:** 어색한 `flame-pilar` 이미지 제거 후 **코드로 그린 네온 화염분수**(바닥에서 솟는, 길이 3종). 추가로 **구정물/오염수(회색·녹색) 장애물** 1종.
- **Why:** 이미지 불기둥이 정적·이질적. 코드 드로우는 일렁임/길이 변주가 자유롭고 톤 일관.
- **경계:** 렌더 전용 — 충돌 박스는 sim의 `o.h` 직사각형 그대로(버전 무관). 텍스처 대신 Graphics로 그림.
- **주의:** 길이 3종은 높이 밴드(`pickObstacleType`)와 매핑. 오염수는 새 타입 키 + 연기 프로파일.

### 2-2. 봇 콜드스타트 수정 + 난이도별 로컬 생성(최대 8)

- **What:**
  1. 봇이 안 도는 문제 수정 — 현재 `uploadBotColdStart`는 Supabase 원격에만, 그것도 다음 세션부터 반영. **로컬(**`saveRun`**)에도 시딩**해 현재 판부터 보이게.
  2. 기록 없을 때 **최대 8개** 봇 생성, 난이도 분포: **초반 사망 / 중거리 사망 / 1만m+ 생존** 구간을 의도적으로 섞기.
- **Why:** 신규 유저 첫 판부터 경쟁 필드 보장(콜드스타트). 난이도 스펙트럼이 있어야 추월의 재미가 산다.
- **경계:** 봇은 입력 로그 재생이라 결정론/버전과 무관(sim 코드는 안 건드림). `botRecorder` 프로파일 확장 + 시딩 경로.
- **검증 필요:** 현재 밸런스로 봇이 1만m+까지 생존 가능한지(프로파일 케이던스 튜닝). 불가하면 목표 거리 하향 조정.

## Tier 3 — 중간~큼 (sim 변경 = SIM_VERSION 업, 묶어서 처리)

### 3-1. 피버 회복량 하향

- **What:** 피버 중 탭 회복(`FEVER_TAP_HEAL`, `src/sim/constants.ts`)을 낮춰 난이도 상향.
- **Why:** 현재 피버 중 체력 회복이 과해 난이도가 너무 쉽다.
- **경계:** **sim 변경 →** `SIM_VERSION` **업 → 기존 고스트/리더보드 무효화.** 변경 자체는 1줄이나 비용은 버전 업.
- **권장:** 다른 sim 튜닝(밸런스 패치)과 **한 번에 묶어** 버전 업 횟수를 줄인다.

## Tier 4 — 큼 (데이터 구조/신규 시스템, 백엔드·론칭 의존)

### 4-1. 닉네임 + 플레이어 고유 id (데이터 구조)

- **현황:** **아직 없음.** `InputLog = { seed, events }`, 원격 `ghost_runs = { seed, sim_version, distance, log, is_bot }` — 닉네임/플레이어 식별자 컬럼이 없다.
- **What:** 시작 시 닉네임 입력 → 레코드에 `nickname`(+ 익명 `playerId`/uuid) 저장. 랭킹/말풍선/최종 라이벌에 노출.
- **Why:** 크로스유저 경쟁의 정체성. _남이 봐야_ 의미 → 백엔드/리더보드와 함께 빛난다.
- **경계:** 메타데이터라 sim 무관(버전 업 없음). 단 스키마 변경(클라 로컬 + Supabase 컬럼) + 입력 UI.
- **Forward-design:** `parseLog`은 필드 없으면 관대하게(기본값) → 가법적 추가, 기존 고스트 안 깨짐. (TODOS.md "Forward-design 스키마 슬롯"과 연계)

### 4-2. 거리 기반 보상 → 아이템 구매(인앱 경제)

- **What:** 최종 달린 거리만큼 인앱 재화 적립 → 아이템(부스터/캐릭터) 구매에 사용.
- **Why:** 리텐션/순환 루프. 단 **닫힌 인앱 경제 + 검증(이상치 필터/리플레이)**이 선행.
- **경계:** 신규 시스템(장부·상점·적립 규칙). 대형. **론칭 후 데이터 기반.** TODOS.md "토큰 경제/부스터" 섹션과 통합.

### 4-3. 랭킹 에셋 + UI 테마 일관성

- **What:** 랭킹 패널을 더 눈에 띄게(네온 테두리/배지 에셋), 전체 UI 폰트·색을 synthwave 팔레트로 통일.
- **Why:** 가독성·완성도. 현재 코드 Rectangle/기본 폰트라 톤이 겉돈다.
- **경계:** 에셋 제작 + 폰트 적용. 렌더 전용(버전 무관)이나 에셋 단계 작업량. TODOS.md "UI 에셋 & 폰트/색상 폴리시"와 동일 축.

## 2026-06-28 추가 요청 묶음 (8건)

> 경계 원칙 동일: 렌더/연출 = 버전 무관 즉시 적용 / 에셋 = 프롬프트만 문서화 후 대기.
> 상태: ✅적용 / 📝프롬프트·문서화(에셋 대기) / 🔧코드 스톱갭 적용(에셋 나오면 교체)

### A-1. 초록색 가스(오염수 분수) 제거 — ✅적용

- **What:** `code-sludge`(녹색 분수)가 "장애물처럼 안 보인다"는 피드백 → 스폰 풀(`OBS_MID`)에서 제거.
- **경계:** 렌더 전용. `drawSludgeFountain`/스모크 프로파일은 재테마 대비 코드만 남김(미선택).

### A-2. 불꽃 장애물 상단 원형 글로우 제거 — ✅적용

- **What:** `drawObstacleSmoke`의 `p.fire` 분기에서 꼭대기 깜빡이는 원형 코어(`0xffd27a` fillCircle) 삭제.
- **Why:** 불기둥 위에 뜬 동그란 빛이 어색. 밑동 베이스 글로우 + 불혀(`drawFlameFountain`)로 역동성 유지.

### A-3. 랭킹 패널 "무조건 YOU" 버그 — ✅적용

- **What:** 고스트 패널 라벨을 항상 `G#`로. 슬롯0(실시간 플레이어)만 YOU.
- **원인:** 원격이 비면 `ghostsAreOwnRecords=true`라 봇/타기록까지 전부 YOU로 표기됐음.
- **잔여(📝):** 내 과거 기록만 'YOU'로 구분하려면 **레코드별 playerId 식별** 필요 → Forward-design `meta.playerId` 슬롯(4-1 닉네임과 함께). 그 전까지 경쟁자는 전부 G#.

### A-4. 2단 점프 ~40° 추가 기울기 — 🔧코드 스톱갭 적용

- **What:** 공중에서 `jumpsUsed>=2`면 플레이어 스프라이트를 -40° 보간 회전(백플립 느낌). 착지/재시작 시 0° 복원.
- **경계:** 렌더 전용(sim 읽기만). 전용 에셋 `player-jump2`(asset-guide §5.1 ★) 나오면 텍스처 스왑으로 교체.

### A-5. 달리는 고스트 후드 유령 컨셉 통일 — 📝프롬프트(asset-guide §5.2A 갱신)

- **What:** 현재 "맨몸 러너"를 후드 쓴 유령으로 → 후드 라이더(주인공)와 세계관 통일하되 실루엣은 구분.
- **경계:** 신규 스프라이트시트 필요. 프롬프트만 갱신, 적용은 에셋 단계.

### A-6. 게임 UI 에셋 (hp바·랭킹패널·튜토리얼·게임오버) — 📝프롬프트(asset-guide §5.7~5.8)

- **What:** 코드 Rectangle UI → 네온 패널 에셋. hp-bar는 §5.7 기존, 랭킹/튜토리얼/게임오버는 §5.8 신규 프롬프트.
- **경계:** 렌더 전용(버전 무관)이나 에셋 제작 단계. TODOS "UI 에셋 & 폰트/색상 폴리시"와 동일 축.

### A-7. 튜토리얼 영상(≤2초) + is_replay 1회 노출 — 📝문서화(asset-guide §6)

- **What:** 인트로/튜토리얼 영상은 2초 이내. **첫 진입 1회만** 노출(재시도/리플레이엔 skip).
- **구현 메모:** 영상 에셋 미존재 → 문서화만. 노출 게이트는 기존 `ga:onboarded` 플래그 패턴 재사용 가능(재시도=is_retry는 skip). 영상 나오면 시작 오버레이에 1회 재생 훅 추가.

### A-8. 아포칼립스 장애물 추가 — 📝프롬프트 기존 + 적용 대기

## Tier 5 — 코어 확장 후보 (🔴 보류 · 별도 판단 필요)

> 여기는 "지금 적용"이 아니라 **코어 컨셉을 바꿀지 말지 결정해야 하는** 큰 변경. sim을 건드려
> `SIM_VERSION`이 올라가고 **기존 고스트/리더보드 기록이 비호환**이 되므로, 렌더 폴리시들과 비용
> 차원이 다르다. 론칭 전이라 기록 무효화는 무료지만, "원탭 코어" 철학을 바꾸는 의사결정이 선결.

### 5-1. 2층(멀티레인) 길 — 점프로 상단 진입 / 슬라이드로 하강 🔴 (보류 강화)

- **⚠️ 보류 사유(2026-06-28):** **화면이 작아** 2개 레인을 세로로 나누면 캐릭터·장애물이 더 작아지고
  레인 간 거리가 좁아져 **가독성·조작 난이도 우려가 큼**. 최우선(위 4개)에서 제외. 화면/뷰포트
  전략(세로 확장, 줌 등)이 정해지기 전엔 착수하지 않는다.
- **What:** 하단 레인(기본) 위에 **상단 레인(2층 길)** 을 추가. 점프로 상단에 올라타고, 하강 입력으로
  내려온다. **레인별로 장애물을 따로 스폰**해, 어느 층에 있느냐 + 전환 타이밍이 회피 단서가 된다.
- **Why:** 회피 축이 "점프 1개"에서 "위/아래 레인 선택 + 타이밍"으로 늘어 **전략성·다이나믹**이 커진다.
  세기말 무너진 고가도로/육교 이미지와도 잘 맞는 서사.
- **경계(왜 비싼가):**
  - **sim 변경 필수** — 레인 상태(`lane: 0|1`), 레인별 충돌 판정, **하강(슬라이드/덕)이라는 두 번째
    입력**을 `src/sim/`에 넣어야 함 → `SIM_VERSION` **범프** → **기존 입력 로그(고스트/봇) 재생 비호환**
    (과거 로그를 새 물리로 돌리면 결과가 달라짐. 마이그레이션 불가, 기록 리셋 전제).
  - **입력 UX 재설계** — 현 코어는 _탭 1개_(`DESIGN.md`). 하강을 (a) 자동 낙하 / (b) 탭-홀드 /
    (c) 별도 영역 탭·스와이프 중 무엇으로 둘지 = **원탭 철학 유지 여부**가 선결 질문.
  - **밸런스 재튜닝** — 점프 높이/중력/스폰 간격(`INTERVAL_`\*)이 단일 레인 기준으로 잡혀 있어
    레인 전환 비행시간·상단 장애물 난이도 곡선을 새로 설계해야 함.
  - **에셋(선택)** — 상단 도로/플랫폼, 레인 전환 연출. 없으면 코드 드로우로 시작 가능.
- **선결 질문 (적용 판단 시 먼저 답):**
  1. 하강을 "자동 낙하"로 둘까, "명시적 슬라이드 입력"으로 둘까? (원탭 코어 유지 여부)
  2. 상단/하단 동시에 장애물을 둘까, 한 번에 한 레인만 둘까? (난이도·가독성)
  3. 고스트도 레인 정보를 기록·재생해야 함 → 기록 스키마(`InputLog`) 확장 범위.
- **의존성:** 코어 컨셉 확정(원탭 vs 멀티입력). 적용 시 다른 sim 튜닝(3-x)과 **한 번에 묶어** 버전 범프.
