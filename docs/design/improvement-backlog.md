# 개선 백로그 (비용 오름차순)

> 2026-06-27 정리. 14개 요청을 **비용(작업량 + 위험도)** 기준으로 묶었다.
> 핵심 경계: **sim 변경 = `SIM_VERSION` 업 = 기존 고스트/리더보드 무효화**(론칭 전이라 무료지만 묶어서 처리). **렌더 변경은 버전 무관, 언제든 자유.**
>
> 적용은 위(싼 것)부터 하나씩.

---

## Tier 0 — 거의 공짜 (렌더/상수, sim 무관, 결정 불필요)

### 0-1. 고스트 8명 제한 ✅ 적용 후보 1순위
- **What:** `GHOST_TOP_N` 15 → 8. 저장·원격 쿼리·렌더 풀 모두 이 상수를 공유하므로 한 곳만 바꾸면 됨.
- **Why:** 화면이 8명 정도일 때 군집 가독성이 좋고, 봇 8개 정책(0-? / Tier2)과도 수가 맞는다.
- **경계:** `src/ghostStore.ts` 상수 1줄. sim 무관 → 버전 업 없음.
- **주의:** `GHOST_TOP_N` 하드코딩 테스트가 있으면 같이 수정.

### 0-2. 오글거리는 랜덤 말풍선
- **What:** 검은 사각 말풍선 + 흰 글씨를 일정 간격마다 랜덤으로 표시.
  - 시작 문구 3종: `지금 내가 걷는 길은 이전에 {n}명의 러너들이 달린 길이지`(n=현재 고스트 수),
    `너와 함께라면 지구 끝까지라도 가보겠어!`, `하루 종일 할 수도 있어`.
- **Why:** 세계관·정서 몰입. 저비용 고효과 분위기 장치.
- **경계:** 렌더 전용(`GameScene`) — 타이머 + 텍스트 풀. sim 무관.

### 0-3. 고스트 점프 시 프레임 정지
- **What:** 고스트가 공중(`g.sim.state.player.y > 임계값`)일 때 `ghost-run` 애니를 멈추고 고정 점프 프레임으로.
- **Why:** 지금은 공중에서도 달리는 모션이라 어색. 플레이어 점프 컷과 일관.
- **경계:** 렌더 전용 — 고스트 스프라이트 anim pause/frame. sim 무관.

## Tier 1 — 저렴 (렌더 전용, 약간의 로직/연출)

### 1-1. 고스트가 죽을 때 "제치는" 이모션
- **What:** 고스트가 기록 끝(=쓰러짐)에 도달할 때, 그 위에 이모지/말풍선 이모션 표시(예: 손 흔들기, 😵, "먼저 갈게~").
- **Why:** 이미 `고스트 제침!` 팝업은 있으나, *그 고스트 머리 위*에 붙는 연출이 추월의 쾌감을 키운다.
- **경계:** 렌더 전용 — collapse 트리거 시점에 이모션 스프라이트/텍스트. 0-2 말풍선 인프라 재사용 가능.

### 1-2. 재시작 시 고스트 정보 최신 재요청
- **What:** "탭하여 재시작" 시 캐시(`this.remoteRuns`)가 아니라 **새로 fetch한 원격 기록**으로 고스트 구성.
- **Why:** 현재 `startRun`은 직전 캐시로 즉시 고스트를 깔고 백그라운드 갱신만 함 → 방금 올라온 남의 기록이 다음 판에 안 보일 수 있음.
- **경계:** 렌더/네트워크 전용. 비동기 순서(로딩 중 임시 고스트 → 도착 시 교체) 처리 필요. sim 무관.

### 1-3. 오토바이 네온 트레일(속도감)
- **What:** 오토바이 뒤에 하늘색 네온 라인/도트 잔상이 반짝이며 따라붙는 이펙트.
- **Why:** 속도감/스타일 강화. 피버 때 강하게 하면 보상감↑.
- **경계:** 렌더 전용 — Graphics/파티클. sim 무관. 속도(`s.speed`)에 비례시키면 자연스러움.

### 1-4. 2단 점프 80도 꺾임 — 에셋 프롬프트
- **What:** 2단 점프 시 ~80도로 더 꺾인 라이더 컷 에셋 **프롬프트만** 작성(`asset-guide.md`).
- **Why:** 2단 점프의 타격감/역동성. 프롬프트는 거의 공짜, 에셋 적용(텍스처 스왑)도 소규모.
- **경계:** 프롬프트=문서. 적용 시 렌더 전용(점프 상태별 텍스처 분기에 컷 추가).

## Tier 2 — 중간 (작업량 있음, 일부 결정/검증 필요)

### 2-1. 불기둥 → 코드 드로우 화염분수 + 오염수 장애물
- **What:** 어색한 `flame-pilar` 이미지 제거 후 **코드로 그린 네온 화염분수**(바닥에서 솟는, 길이 3종). 추가로 **구정물/오염수(회색·녹색) 장애물** 1종.
- **Why:** 이미지 불기둥이 정적·이질적. 코드 드로우는 일렁임/길이 변주가 자유롭고 톤 일관.
- **경계:** 렌더 전용 — 충돌 박스는 sim의 `o.h` 직사각형 그대로(버전 무관). 텍스처 대신 Graphics로 그림.
- **주의:** 길이 3종은 높이 밴드(`pickObstacleType`)와 매핑. 오염수는 새 타입 키 + 연기 프로파일.

### 2-2. 봇 콜드스타트 수정 + 난이도별 로컬 생성(최대 8)
- **What:**
  1. 봇이 안 도는 문제 수정 — 현재 `uploadBotColdStart`는 Supabase 원격에만, 그것도 다음 세션부터 반영. **로컬(`saveRun`)에도 시딩**해 현재 판부터 보이게.
  2. 기록 없을 때 **최대 8개** 봇 생성, 난이도 분포: **초반 사망 / 중거리 사망 / 1만m+ 생존** 구간을 의도적으로 섞기.
- **Why:** 신규 유저 첫 판부터 경쟁 필드 보장(콜드스타트). 난이도 스펙트럼이 있어야 추월의 재미가 산다.
- **경계:** 봇은 입력 로그 재생이라 결정론/버전과 무관(sim 코드는 안 건드림). `botRecorder` 프로파일 확장 + 시딩 경로.
- **검증 필요:** 현재 밸런스로 봇이 1만m+까지 생존 가능한지(프로파일 케이던스 튜닝). 불가하면 목표 거리 하향 조정.

## Tier 3 — 중간~큼 (sim 변경 = SIM_VERSION 업, 묶어서 처리)

### 3-1. 피버 회복량 하향
- **What:** 피버 중 탭 회복(`FEVER_TAP_HEAL`, `src/sim/constants.ts`)을 낮춰 난이도 상향.
- **Why:** 현재 피버 중 체력 회복이 과해 난이도가 너무 쉽다.
- **경계:** **sim 변경 → `SIM_VERSION` 업 → 기존 고스트/리더보드 무효화.** 변경 자체는 1줄이나 비용은 버전 업.
- **권장:** 다른 sim 튜닝(밸런스 패치)과 **한 번에 묶어** 버전 업 횟수를 줄인다.

## Tier 4 — 큼 (데이터 구조/신규 시스템, 백엔드·론칭 의존)

### 4-1. 닉네임 + 플레이어 고유 id (데이터 구조)
- **현황:** **아직 없음.** `InputLog = { seed, events }`, 원격 `ghost_runs = { seed, sim_version, distance, log, is_bot }` — 닉네임/플레이어 식별자 컬럼이 없다.
- **What:** 시작 시 닉네임 입력 → 레코드에 `nickname`(+ 익명 `playerId`/uuid) 저장. 랭킹/말풍선/최종 라이벌에 노출.
- **Why:** 크로스유저 경쟁의 정체성. *남이 봐야* 의미 → 백엔드/리더보드와 함께 빛난다.
- **경계:** 메타데이터라 sim 무관(버전 업 없음). 단 스키마 변경(클라 로컬 + Supabase 컬럼) + 입력 UI.
- **Forward-design:** `parseLog`은 필드 없으면 관대하게(기본값) → 가법적 추가, 기존 고스트 안 깨짐. (TODOS.md "Forward-design 스키마 슬롯"과 연계)

### 4-2. 거리 기반 보상 → 아이템 구매(인앱 경제)
- **What:** 최종 달린 거리만큼 인앱 재화 적립 → 아이템(부스터/캐릭터) 구매에 사용.
- **Why:** 리텐션/순환 루프. 단 **닫힌 인앱 경제 + 검증(이상치 필터/리플레이)**이 선행.
- **경계:** 신규 시스템(장부·상점·적립 규칙). 대형. **론칭 후 데이터 기반.** TODOS.md "토큰 경제/부스터" 섹션과 통합.

### 4-3. 랭킹 에셋 + UI 테마 일관성
- **What:** 랭킹 패널을 더 눈에 띄게(네온 테두리/배지 에셋), 전체 UI 폰트·색을 synthwave 팔레트로 통일.
- **Why:** 가독성·완성도. 현재 코드 Rectangle/기본 폰트라 톤이 겉돈다.
- **경계:** 에셋 제작 + 폰트 적용. 렌더 전용(버전 무관)이나 에셋 단계 작업량. TODOS.md "UI 에셋 & 폰트/색상 폴리시"와 동일 축.

## 권장 적용 순서

1. **0-1 고스트 8명** → 2. **0-2 말풍선** → 3. **0-3 고스트 점프 프레임** (전부 렌더/상수, 무위험)
4. **1-1 제치는 이모션** → 5. **1-2 재시작 재요청** → 6. **1-3 네온 트레일** → 7. **1-4 점프 컷 프롬프트**
8. **2-1 화염분수+오염수** → 9. **2-2 봇 수정/난이도**
10. **3-1 피버 회복 하향**(다른 sim 튜닝과 묶기)
11. 이후 **4-x**(닉네임 → 보상 → UI 에셋)은 백엔드/론칭 단계.
